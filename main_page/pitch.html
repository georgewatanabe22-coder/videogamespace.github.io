CISGAME 420 Team 1 Pitch Document

Game Concept

Our final project is a game portfolio website that showcases several games created by Team 1. The website presents each member’s game, screenshots, media, and project information in one shared online space. The overall concept is to create a polished game showcase with a futuristic arcade style.

Target Audience

The target audience is casual game players, students, and anyone interested in simple web and mobile-style games. The games are designed to be easy to understand, quick to play, and accessible through a web browser.

Core Mechanics

Each game focuses on simple interactive mechanics, such as movement, dodging obstacles, reacting quickly, and improving score or survival time. The website also uses interactive elements such as navigation, audio controls, hover effects, and project links.

Monetization Strategy

If developed further, the games could use simple monetization strategies such as optional ads, cosmetic upgrades, or free-to-play browser access with future downloadable versions. For this class project, the main goal is demonstration and portfolio presentation rather than active monetization.

Development Timeline

  • Concept and planning, choose project structure and team roles.
  • Game development, create or refine individual games.
  • Website development, build the shared homepage and project pages.
  • Media and polish, add screenshots, audio, styling, and navigation.
  • Testing and submission, verify links, layout, and hosted website functionality.

Marketing Strategy

The game portfolio could be promoted through a shared website link, class presentations, social media, and game hosting platforms. The strongest marketing point is that the website allows visitors to view multiple projects in one place while seeing each team member’s contribution.
